# AGENTS.md

本文件为在本仓库工作的 AI 编码助手提供上下文。修改代码前请先阅读本文档与 [docs/DEVELOPMENT.md](docs/DEVELOPMENT.md)。

## 项目是什么

**DesignPatterns** 是一个 .NET 设计模式工具库：

- **运行时**：轻量、可组合的 primitives（责任链、策略、工厂注册表、Singleton 特性等），组合优于继承。
- **编译期**：`DesignPatterns.SourceGenerators` 提供增量源生成器与 `DP###` 诊断；CodeFix / Completion 尚未实现。

不要把它做成「23 种模式的厚重框架」或 MediatR/Polly 的替代品。

## 实现新功能时的规则

### 运行时 API

- 保持 **最小依赖**；不要把 `Microsoft.Extensions.DependencyInjection` 拉进 Core。
- 公共 API 使用 XML 注释；异步方法以 `Async` 结尾，接受 `CancellationToken`。
- 优先接口 + 扩展方法 + 委托，避免深层 `abstract` 基类。
- 解析/查找用 `Try*` 或明确异常，避免无声失败。
- 命名空间按 GoF 分类：`DesignPatterns.Behavioral`、`DesignPatterns.Creational`、`DesignPatterns.Structural`。

### 依赖注入集成（规划中，独立扩展包）

涉及 DI 时，优先支持以下容器（按优先级排列）：

1. **Autofac**
2. **Microsoft.Extensions.DependencyInjection (MSDI)**
3. **DryIoc**

Core **不引用** 任何 DI 包；扩展包命名如 `DesignPatterns.Autofac`、`DesignPatterns.Extensions.DependencyInjection`。

### Roslyn（`DesignPatterns.SourceGenerators`）

- 生成器：**`IIncrementalGenerator`** + `ForAttributeWithMetadataName`。
- 项目属性：`IsRoslynComponent=true`，`EnforceExtendedAnalyzerRules=true`，`IncludeBuildOutput=false`。
- 只生成**注册表、排序管道、强类型 key 常量、Lazy Instance** 等胶水；不生成完整业务状态机。
- 优先生成强类型常量（如 `PaymentStrategyKeys`）；CompletionProvider 维护成本高，非必要不新增。
- NuGet 元包将生成器 DLL 放入 `analyzers/dotnet/cs/`。

#### 源生成器：优先 `SyntaxFactory`，避免字符串拼接

编写生成代码时，**尽可能使用 Roslyn 语法 API**（`SyntaxFactory`、`CompilationUnitSyntax` 等）构建语法树，再 `NormalizeWhitespace()` 后 `AddSource`；**不要**用 C# 源码字符串拼接生成逻辑。

`CompilationUnit` 的 `using` 必须在 `AddMembers` **之前**添加，否则可能触发 CS8802（与 top-level statements 冲突）。

仅在以下情况可例外（PR 中说明理由）：

- 固定的文件头注释（`// <auto-generated />`）。
- 极小的、无分支的常量片段，且已有测试覆盖。

参考实现：`Syntax/StrategySyntaxFactory.cs`、`Syntax/HandlerPipelineSyntaxFactory.cs`、`Syntax/SingletonSyntaxFactory.cs`、`Syntax/CompositeSyntaxFactory.cs`。

### 文件与命名

- 新项目名称与文件夹 PascalCase，与现有风格一致。
- 泛型 Attribute（如 `[RegisterStrategy<T>]`、`[HandlerOrder<T>]`）用 `#if NET7_0_OR_GREATER` 条件编译；非泛型版本供 `netstandard2.0` 使用。

## 不要做

- 不要提交 `bin/`、`obj/`、`.vs/`（已在 `.gitignore`）。
- 不要未经用户要求执行 `git commit` / `git push`。
- 不要在 Core 中引入重量级第三方框架。
- 不要用反射扫描 AppDomain 替代编译期注册。
- 不要为教学目的添加大段注释替代 `samples/` 示例。
- 不要过度工程：避免无意义抽象层；Strategy/Factory 共用注册表抽象可延后。
- 不要在源生成器中用字符串拼接生成 C# 源码（见 SyntaxFactory 规则）。
